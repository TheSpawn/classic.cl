Manual de Usuario

CMS multi-sitio de Classic Producciones. Este manual explica cada seccion, cada campo y como configurar el contenido de los sitios web del ecosistema.

Acceso al CMS

URL: https://cms.classic.cl/login

Email
Email registrado del usuario administrador.
Ej: admin@classic.cl
Contrasena
Contrasena del usuario. Se puede cambiar desde Mi Perfil.
Roles: SUPERADMIN (acceso total), ADMIN (gestiona contenido de sitios asignados), EDITOR (solo edita contenido).

Dashboard

Pantalla principal que muestra los contadores de cada modulo para el sitio activo: eventos, galerias, alianzas, documentos, partners, precios e hitos. Haz clic en cualquier tarjeta para ir al modulo.

Selector de sitio

En la parte superior del sidebar hay un selector desplegable. Debes seleccionar un sitio antes de gestionar contenido. Todo el contenido que crees (eventos, alianzas, precios, etc.) queda asociado al sitio seleccionado.

Sitios disponibles: Cheerleader Classic, Dance Classic, Gym Classic (u otros que se agreguen).

Mi Perfil

Permite ver tus datos y cambiar tu contrasena. Accesible desde el sidebar o el boton "Salir" en la barra superior.

Contrasena actual
Tu contrasena actual para verificar identidad.
Nueva contrasena
Minimo 6 caracteres.

Eventos

Los eventos son el contenido principal. Cada evento aparece en la pagina de eventos del sitio web y tiene su propia pagina de detalle.

Campos del formulario

Titulo obligatorio
Nombre completo del evento.
Ej: Classic Super Nationals Cheer 2026
Slug obligatorio, unico
Identificador para la URL. Solo letras minusculas, numeros y guiones. Sin espacios ni caracteres especiales.
Ej: super-nationals-2026
Fecha
Fecha del evento en formato YYYY-MM-DD (el selector de fecha lo maneja automaticamente).
Hora
Hora de inicio del evento.
Ej: 09:00
Ubicacion
Direccion o ciudad donde se realiza el evento. Aparece en el sidebar del detalle como "Direccion".
Ej: Avenida Grecia #8787, Penalolen
Venue
Nombre del recinto. Aparece en el sidebar del detalle como "Recinto".
Ej: Centro Cultural Chimkowe
Estado
Define el estado visual del evento en el sitio web.
PRONTO = badge "Proximamente" | ABIERTO = badge "Inscripciones abiertas" + boton inscripcion | CERRADO = badge "Cerrado"
Icono
Icono de Bootstrap Icons que representa al evento. Usa el boton para abrir la guia visual de iconos.
Ej: bi-trophy-fill
Imagen 1920x800 px recomendado
Imagen principal del evento. Se usa como fondo del hero en la pagina de detalle. Formato WebP recomendado.
Video
URL de YouTube o Vimeo, o subir un archivo MP4 (max 50MB, H.264, 1280x720). Se muestra en la pagina de detalle del evento.
Ej: https://www.youtube.com/watch?v=XXXXXXXXXXX
Descripcion corta texto plano
Resumen breve del evento. Se muestra en tarjetas de listado, hero y meta description para SEO. No acepta HTML.
Descripcion completa acepta HTML
Texto completo del evento. Se muestra en la seccion "Acerca del Evento". Acepta etiquetas HTML: <strong>, <br>, <p>, <ul><li>.
Vende entradas
Define si el evento tiene venta de entradas al publico. Si: el sitio web muestra el boton "Comprar Entradas". No: el boton no aparece (usar para capacitaciones, clinicas o cursos donde solo hay inscripcion/registro).
Competencia = Si | Capacitacion o Clinica = No
Orden
Numero para ordenar los eventos en el listado. Menor numero aparece primero.
Activo
Si = visible en el sitio web. No = oculto.

Meta datos (info icons)

Son iconos informativos que aparecen debajo de la descripcion corta en la tarjeta del evento. Cada meta dato tiene un icono y un texto.

Icono
Clase de Font Awesome para el icono.
Ej: fa-solid fa-calendar, fa-solid fa-building, fa-solid fa-globe
Texto
Texto informativo breve.
Ej: Centro Cultural Chimkowe | School & All Star | Clasificacion internacional
Si no agregas meta datos: El sistema los genera automaticamente desde los campos fecha, hora y ubicacion del evento.
Portal corporativo: Los eventos activos de todos los sitios aparecen automaticamente en la seccion "Proximos Eventos" del portal classic.cl. Al hacer clic, se abre la pagina del evento en el sitio de la marca correspondiente (nueva ventana).

Highlights

Lista de puntos destacados que aparecen en la seccion "Que Incluye" del detalle del evento, cada uno con un check verde.

Texto del highlight
Texto corto y directo sobre lo que incluye o destaca del evento.
Ej: Divisiones School y All Star | Normativa IASF oficial | Clasificacion a The Summit

Precios

Los precios estan vinculados a un evento especifico. Representan los tiers de inscripcion (Anticipado, On-Time, Final).

Evento obligatorio
Selecciona a cual evento pertenece este precio.
Nombre del tier obligatorio
Nombre del periodo de precio.
Ej: Anticipado, On-Time, Final
Monto obligatorio, numero entero
Valor en la moneda seleccionada. Sin puntos ni comas.
Ej: 35000 (se mostrara como $35.000)
Moneda
CLP (Pesos chilenos) o USD (Dolares).
Fecha inicio / Fecha fin
Periodo de vigencia del precio. El sitio web muestra automaticamente si el periodo esta activo, proximo o cerrado segun la fecha actual.
Caracteristicas
Lista de lo que incluye este precio. Se muestra como bullets en la tarjeta del precio.
Ej: Inscripcion al campeonato | Acceso a instalaciones | Certificado de participacion
Ahorro automatico: El sistema calcula automaticamente el ahorro respecto al precio mas alto y lo muestra como badge.

API REST

Los sitios web frontend consumen los datos del CMS a traves de la API en api.classic.cl.

Autenticacion

Todas las peticiones requieren header Authorization: Bearer {API_SECRET_KEY}

Endpoints

GET /v1/sitios
Lista de sitios activos.
GET /v1/{sitio}/eventos
Eventos del sitio con meta datos, highlights y campo vende_entradas (1 = si, 0 = no).
GET /v1/{sitio}/eventos/{slug}
Detalle de un evento con precios incluidos y campo vende_entradas.
GET /v1/{sitio}/precios
Precios del sitio. Filtrar por evento: ?evento={slug}
GET /v1/{sitio}/alianzas
Alianzas internacionales.
GET /v1/{sitio}/galeria
Albumes de galeria.
GET /v1/{sitio}/documentos
Documentos descargables.
GET /v1/{sitio}/partners
Partners/sponsors.
GET /v1/{sitio}/contenido
Contenido generico agrupado por seccion.
GET /v1/{sitio}/hitos
Timeline de hitos.

Donde {sitio} es el slug: cheerleader, dance, gym.

Cache

Los frontends cachean las respuestas de la API por 30 segundos (desarrollo) o 5 minutos (produccion). Si haces un cambio en el CMS y no se refleja inmediatamente, espera a que expire el cache.

Forzar limpieza: POST /v1/cache/purgar con Bearer token limpia el cache del lado del CMS.

Archivos subidos

Todos los archivos subidos (imagenes, documentos, videos) se almacenan en el servidor del CMS en la ruta:

public/uploads/{slug-sitio}/{modulo}/

Ej: uploads/cheerleader/eventos/, uploads/dance/alianzas/

Los frontends acceden a estos archivos a traves de la URL completa del CMS (ej: https://classic.cl/uploads/cheerleader/eventos/foto.webp).
